# Audit Log Security Contract

## 신뢰 경계
- 감사 이벤트의 소스는 각 서비스 내부다.
- 외부 입력을 그대로 감사 이벤트로 저장하지 않는다.
- 감사 저장소는 내부망 또는 신뢰 가능한 운영 경로에서만 접근한다.

## 데이터 보호
- 토큰, 비밀번호, 원문 쿠키, 개인정보 원문은 저장하지 않는다.
- 필요 시 `metadata`는 마스킹/해시 처리 후 저장한다.
- 사용자 식별자는 최소 식별자만 남기고, 고위험 데이터는 별도 접근 제어를 둔다.

## 이벤트 위변조 방지
- 이벤트는 append-only 방식으로 저장한다.
- 가능한 경우 eventId, timestamp, source service를 포함해 체인을 검증할 수 있어야 한다.
- 감사 이벤트 수집 실패는 반드시 별도 운영 경보로 남긴다.

## 접근 정책
- 감사 조회 API 또는 대시보드는 내부 관리자/보안 운영자만 접근한다.
- 서비스별 운영자는 자기 서비스 범위만 조회할 수 있어야 한다.
- 민감 이벤트는 보관 기간과 접근 범위를 더 엄격하게 제한할 수 있다.
