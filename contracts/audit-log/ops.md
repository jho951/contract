# Audit Log Operations Contract

## 운영 책임
- 감사 이벤트 발행 실패는 각 서비스의 운영 알람 대상이다.
- `audit-log` 저장 실패가 서비스 주요 흐름을 막을지 여부는 서비스별로 정한다.
- 고위험 이벤트는 가능한 한 동기 또는 준동기 방식으로 전달한다.

## 보존 정책
| 범위 | 기본 권장 |
| --- | --- |
| 인증/인가 이벤트 | 장기 보관 가능 |
| 관리자 이벤트 | 장기 보관 우선 |
| 일반 사용자 이벤트 | 보존 기간을 정책으로 제한 가능 |
| 캐시 이벤트 | 운영 목적에 한해 단기 보관 가능 |

## 운영 점검
- 이벤트 유실률을 점검한다.
- 수집 지연을 확인한다.
- 서비스별 발행량 급증을 추적한다.
- 보안 감사용 검색 가능성을 주기적으로 검증한다.

## 실패 처리
- 감사 저장 실패가 비즈니스 요청 실패로 이어질지 여부는 서비스 계약에서 정의한다.
- 최소한의 보안 이벤트는 실패 시 별도 재시도 경로를 둔다.
